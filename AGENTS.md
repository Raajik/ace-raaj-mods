## 4. Repository & Deployment

### 4.1 Mod Structure
Each folder = deployable mod:
`Meta.json`, `Settings.cs`, `Settings.json` (repo template), `*.csproj`

### 4.2 `Settings.json` Rules
- **Test server** (`C:\ACE\Mods\<AssemblyName>\Settings.json`) is the **operator source of truth** — never overwrite on deploy unless explicitly requested.
- Adding new JSON keys: update `Settings.cs` + repo `Settings.json`, then **merge** into test file. `push test`: deploy DLL/Meta only; **preserve** test `Settings.json`.
- **Repo `Settings.json` must match void-test defaults** — compare before shipping.

### 4.3 Deployment Overview
Three environments — see `docs/deployment.md` for full commands and flags:

| Environment | DB | Deploy Command | Notes |
|-------------|----|----------------|-------|
| void-test | `void-test_world` | `bash scripts/deploy-void-test.sh` | Isolated balancing environment |
| wb_test (`C:\ACE\`) | `ace_world` (default) | `bash scripts/deploy-wb-test.sh` | Stops ACE, wipes + copies mods, syncs world |
| Live (`C:\ACE-WB\`) | `wb_ace_world` | Manual copy only | Never touch without "push live" |

Key env vars: `VOID_TEST_SKIP_SQL`, `DEPLOY_WB_TEST_SKIP_WORLD_SYNC`, `ACE_MYSQL_USER`, `ACE_MYSQL_PASSWORD`.

**ModData** (`Server/ModData/<ModName>/`) is **never** wiped by deploy scripts. Path resolves from `Assembly.GetEntryAssembly().Location` (ACE.Server.exe directory), not `Assembly.Location` (empty for plugins).

### 4.4 SQL Workflow
1. **Back up first** — `mysqldump` scoped into `sql-backups/YYYY-MM-DD/`
2. Apply in order: `void-test_world` → `ace_world` (test) → `wb_ace_world` (live)
3. Verify with `SELECT` after each step
4. See `docs/deployment.md` and wiki `operations/SQL Procedures` for full guide

### 4.5 CI / Releases
- Release zips: auto-generated by `.csproj` `ZipOutputPath` in Release config
- GitHub Actions: handles CI builds and releases

## 5. Development Conventions

### 5.1 Harmony Patching
- **Prefer `nameof` targeting** — string-based targets fail silently at runtime.
- Prefix patch methods with `Pre`/`Post`/`Transpiler`. Use `PatchCategory` for grouped unpatch.
- **Patch base `virtual`, not `override`** (e.g., patch `WorldObject.Heartbeat`, not `Creature.Heartbeat`).
- Initialize `Settings` before `base.Start()` in `BasicPatch<>` subclasses.
- **Cross-mod Harmony:** Use `AccessTools.TypeByName("Namespace.Type")` inside static `TargetMethod()`, with `Prepare()` returning `false` if absent.

### 5.2 Cross-Mod Integration (Property IDs)
**Check shared IDs before inventing new ones.** See `docs/properties.md` for the full mapping table.

### 5.3 Cross-Mod Ownership
- **LeyLineLedger** owns ALL bank, vendor commerce, and pricing. No other mod implements vendor price multipliers.
- **QOL `VendorLootRotation`** only regenerates sale stock (Harmony **First**). It is **not** a pricing mod.
- **BetterLootControl** is the single source of truth for all loot tables.
- **AureatePath** owns the server level cap.

### 5.4 Technical Patterns & Gotchas
- **Threading:** Static `HashSet<T>` is not thread-safe. Use `ConcurrentDictionary<T, byte>`. `ThreadSafeRandom.Next(min, max)` is **inclusive on both bounds** — use `Length - 1`.
- **Vendor ordering:** CLEARS inventory → `Priority.First`. ADDS static items → `Priority.Last`.
- **Monetary rewards:** Must use `LeyLineLedgerBankInterop.DepositToBank`. Never give physical trade notes/py real.
- **Full pattern reference:** `docs/patterns.md`
- **Quick gotchas table:** `docs/gotchas.md`

### 5.5 Windblown Custom Trophies
When adding tiered quest trophies (real physical stackables, autoloot to pack):
- Use template at `Windblown/docs/Tiered-Trophy-Template.md`
- Update `UpgradedTrophyWeenieClassIds` in AutoLoot `Settings.json`
- Bulk turn-in: `EnableBulkTurnIn: true` (already in trophy line JSON)
