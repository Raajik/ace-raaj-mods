## 2. Starting Workflow
Always check in this order:
1. `PLAN.md` ‚Äî Master todo index, suggested implementation order, greenfield backlog.
2. `README.md` ‚Äî Mod list, enablement status.
3. Mod-specific `Readme.md` ‚Äî Per-mod docs and configuration notes.
4. **Skills** ‚Äî Load domain-specific skills before touching code.

## 3. Repo Conventions
- **Mod structure:** Each folder = deployable mod containing:
  - `Meta.json` ‚Äî Enable/disable, hot-reload, version. **Do not change `Enabled` without user direction.**
  - `Settings.cs` ‚Äî C# defaults. Use `JsonPropertyName("// ...")` doc band + values band pattern.
  - `Settings.json` ‚Äî Runtime overrides. Safe to edit freely.
  - `*.csproj` ‚Äî Builds to `C:\ACE\Mods\$(AssemblyName)` locally.
- **Harmony patches:** Prefer `nameof` targeting. Prefix patch methods with `Pre`/`Post`/`Transpiler`. Use `PatchCategory` for grouped unpatch.
- **Cross-mod properties:** Check shared IDs before inventing new ones:
  - `FakeFloat 11012` = EnlightenmentPoolBonus (AureatePath / Loremaster / ChallengeModes)
  - `FakeInt 40113` = BuddySpawn tag (Swarmed)
  - `FakeInt 10029` = CreatureExType (Swarmed)

## 4. Agent Permissions
- **DO:** Edit `Settings.json`, fix bugs, tune values, refactor for clarity.
- **DO:** Commit and push when changes are verified and make sense.
- **DO NOT:** Change `Meta.json` enablement without explicit user direction.
- **DO NOT:** Create new mods without confirming scope.

## 5. Build & Deploy Notes
- Build locally via `dotnet build` in each mod folder (outputs to `C:\ACE\Mods\`).
- Release zips are auto-generated by `.csproj` `ZipOutputPath` target in Release config.
- GitHub Actions workflow handles CI builds and releases.

## 6. Releases & Changelogs
- **Every push** to GitHub must include a clear changelog / patch notes.
- Update the feature list and mod descriptions in `README.md` when adding or changing significant functionality.
- Release notes are auto-generated from commits, but hand-written summaries for user-facing changes are strongly preferred.

## 6. Server Logs
- **`C:\ACE\Server\ACE_Log.txt`** is always allowed to be read and cleared if needed for diagnostics.

## 7. Reference Sources
- **`.references/`** contains ACE source/server files, ACRealms, and world database dumps. Check this directory when you need to inspect ACE internals, verify class/method signatures, or reference vanilla behavior.
